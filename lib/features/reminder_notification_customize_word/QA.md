## 2. 通知タイトル（先頭文字）の編集

- [ ] **文字数カウントと8文字の上限**: タイトル入力欄に文字を入力すると「入力文字数/8」のカウンタが表示され、8文字を超えて入力できない（絵文字も1文字としてカウントされる）。入力欄を全て削除すると、カウンタの代わりに赤字の警告文言が表示される

## 4. 表示オプションの切り替え

- [ ] **日付・番号表示の切り替え**: 「日付を表示」「番号を表示」のスイッチを切り替えると、他の操作を挟まず即座に設定が保存され、プレビュー内のサンプル日付（例: 1/7）・ピル番号表示（例: 5番 ~ 8番）の表示/非表示が切り替わる。画面を離れて再度開いても切り替えた状態が保持されている

## 5. エッジケース

- [ ] **未確定の変更は保存されない**: タイトルまたはメッセージ欄に文字を入力したが「完了」を押さずに画面左上の戻るボタンで離脱した場合、変更は保存されず、再度開くと編集前の値に戻っている
- [ ] **実通知への反映確認**: 保存操作のたびにローカル通知の再登録処理が呼ばれるが、実際のリマインダー通知が発火するまで待って文言変更を確認するのはシミュレータ上では時間がかかり不安定なため確認困難。本QAではプレビュー表示上の文言反映確認に留める
