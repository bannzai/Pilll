# before_pill_sheet_group_history QA

過去に破棄・終了したピルシートグループ（服薬履歴）を一覧・詳細表示する閲覧専用画面（`BeforePillSheetGroupHistoryPage`）。編集・削除操作は存在しない。

## 1. 画面表示（前回のピルシートグループがある場合）

- [ ] **画面到達と初期表示**: 記録画面の設定シート（歯車アイコン）→「ピルシートをすべて破棄」→確認ダイアログで「破棄する」を選択して現在のピルシートグループを終了したあと、表示される「＋ ピルシートを追加」から新しいピルシートグループを作成する。この状態で本画面を表示すると、AppBar タイトルに「前回のピルシートグループ」、その下に破棄前グループの開始日〜終了予定日が「yyyy/MM/dd ~ yyyy/MM/dd」形式で表示される
  - Note: 2026-07-03 時点で本画面への遷移導線は `lib/features/record/components/header/record_page_header.dart` 内でコメントアウトされており（TODOコメント `[PillSheetModifiedHistory-V2-BeforePillSheetGroupHistory] 2024-05-01`）、通常のタップ操作では到達できない。動作確認時は当該コメントアウトを一時的に解除する（履歴アイコンボタンを復活させ `Navigator.of(context).push(BeforePillSheetGroupHistoryPageRoute.route())` を呼べるようにする）等でビルドし直してから検証する
- [ ] **表示内容が「前回」のものであることの確認**: 上部の期間表示・ピルシートの服用済みマークが、現在記録画面で使用中の最新グループの内容ではなく、直前に破棄したグループの開始日・終了予定日・服用済み状態と一致していることを確認する（`beforePillSheetGroupProvider` の実装上、最新グループと同じ内容が誤って表示されていないか要注意）
- [ ] **戻る操作**: AppBar 左上の戻るボタン（矢印アイコン）をタップすると前の画面（記録画面）に戻る

## 2. 服用記録の修正履歴表示

- [ ] **修正履歴一覧表示**: 画面下部に、上部で表示中のピルシート期間（開始日〜終了予定日）に該当する服用記録の修正履歴が月別ヘッダー付きで一覧表示される（服用記録・服用取り消し・お休み開始/終了・シート作成などのイベントが該当）
- [ ] **履歴取得エラー時の表示**: 履歴取得に失敗した場合、一覧の代わりに「服用履歴情報の取得に失敗しました。」というエラーメッセージが表示される
  - Note: 意図的な再現が難しい（通常操作では発生しにくい）。機内モード化など疑似的な手段で再現できない場合はスキップしてよい

## 3. 複数ピルシート・ページ切り替え

- [ ] **複数シート時のページ送りと表示更新**: 前回のピルシートグループに複数のピルシートが含まれる場合、横スワイプで次/前のシートに切り替えられ、上部の期間表示と下部の修正履歴一覧が切り替え後のシートの日付範囲に応じて更新される
  - Note: 1グループに複数ピルシートが含まれるのは、1日2錠服用設定（v2）など特定の構成に限られる。単一シートのグループしか用意できない場合はこの項目はスキップする
- [ ] **ドットインジケーターの表示・タップ**: グループ内のピルシートが2つ以上の場合のみ画面下部にドットインジケーターが表示され、任意のドットをタップするとそのページへ直接ジャンプする。ピルシートが1つのみの場合はドットインジケーターが表示されない
