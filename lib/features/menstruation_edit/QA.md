# menstruation_edit QA

## 1. 新規作成（日付範囲ピッカー）

- [x] **新規作成ピッカー表示**: 生理タブの記録フローから新規作成用のピッカーを開くと、ヘルプテキストに「生理開始日を選択」が表示され、開始日・終了日を選択できる
- [x] **選択可能な日付範囲**: 日付範囲ピッカーで選択できるのは2020年1月1日から今日+30日までで、それ以外の日付は選択できない
- [x] **保存で新規記録が作成される**: 開始日・終了日を選んで確定すると新しい生理記録が作成され、「◯月◯日から生理開始で記録しました」のスナックバーが表示される
- [x] **キャンセル時は何も作成されない**: ピッカーを日付未選択のままキャンセルすると、生理記録は作成されない

## 3. 削除

- [x] **削除確認ダイアログの表示**: 「削除」をタップすると「生理期間を削除しますか？」の確認ダイアログが「キャンセル」「削除する」の2ボタンで表示される
- [x] **キャンセルでは削除されない**: 確認ダイアログの「キャンセル」をタップするとダイアログのみ閉じ、記録は削除されず選択シートも表示されたままになる
- [x] **削除実行**: 確認ダイアログの「削除する」をタップすると記録が削除され、「生理期間を削除しました」のスナックバーが表示されたうえで選択シートも閉じる
- [ ] **削除失敗時のエラー表示**: 削除処理が失敗した場合はエラーアラートが表示される。通信断が必要な項目のため通常操作では再現しづらく、Simulatorを機内モード等で通信遮断した状態で削除操作を行うことで代替確認する
  - ⏭️ スキップ: iOS シミュレータの「機内モード」トグルはUI上の表示のみでホストMacのネットワークスタックをそのまま使うため、実際のネットワーク遮断にならず削除失敗を再現できない。ホスト全体のネットワークを切断する方法は他の同時起動中シミュレータ・プロセスへの影響が大きいため今回は実施せず、コードレビューでのみ確認（`menstruation_edit_selection_sheet.dart` の削除処理は `try/catch` で `showErrorAlert(context, e)` を呼ぶ実装になっている）
